# 多无人机区域协同搜索决策模型总体设计方案

## 一、项目目标

开发一个面向城市复杂区域（1–2公里边长）的多无人机协同搜索决策系统。系统作为纯决策层，接收外部环境和无人机状态信息，输出任务分配和路径规划结果。核心目标：

1. 在存在楼宇、障碍物、禁飞区的复杂环境中，协调 5–10 架无人机高效完成区域覆盖搜索
2. 在搜索过程中对发现的目标做出响应（发现者抵近确认）
3. 支持动态环境变化（新障碍物、目标出现）的实时重规划
4. 单步决策延迟控制在 1 秒以内
5. 提供 2D 仿真环境验证算法效果

## 二、系统边界

### 系统负责

- 环境地图管理（栅格化、状态维护）
- 搜索任务生成与分解
- 多机任务分配（拍卖算法）
- 路径规划（A*）
- 动态重规划与冲突检测
- 搜索覆盖率评估
- 2D 仿真与可视化

### 系统不负责

- 无人机飞行控制（姿态、油门、PID）
- 传感器数据处理（图像识别、目标检测算法本身）
- 通信协议与链路管理
- 武器或任何打击控制

系统通过定义好的接口接收外部输入、输出决策结果，与飞控系统解耦。

## 三、场景假设

1. **区域特征**：搜索区域为 1–2 公里边长的矩形或多边形区域，包含建筑物、禁飞区（如学校、政府机关上空）、重点搜索区域（如事发地周边）
2. **无人机编队**：5–10 架同构旋翼无人机，飞行高度统一（二维规划），性能参数相同（速度、续航、传感器覆盖半径）
3. **通信条件**：理想通信，所有无人机可随时与中心决策节点双向通信，无延迟无丢包
4. **决策架构**：集中式，中心节点拥有全局信息，统一做任务分配和路径规划
5. **地图先验**：系统启动时加载完整底图（建筑物轮廓、禁飞区边界、重点区域标注），运行中可动态更新
6. **目标特征**：目标为地面静止或低速移动物体（人员、车辆），出现位置和时间未知
7. **时间约束**：系统以固定周期（默认 1–2 秒）滚动决策，高优先级事件（发现目标、无人机离线）立即触发重规划

### 系统输入

| 输入项 | 内容 | 更新频率 |

| 系统指令 | 启动/暂停/终止搜索、添加重点区域 | 人工触发 |

### 系统输出

| 输出项 | 内容 | 说明 |
|--------|------|------|
| 任务分配表 | 每架无人机当前任务（搜索/确认/返航） | 每周期更新 |

## 五、环境地图建模

### 栅格表示

- 将搜索区域离散化为二维栅格，默认分辨率 10m×10m，支持配置
- 每个栅格单元存储多层属性信息

### 栅格属性层

| 属性层 | 数据类型 | 说明 |
|--------|----------|------|
| 地形类型 | 枚举 | 自由区域 / 障碍物 / 禁飞区 / 重点区域 |
| 可通行性 | 布尔 | 无人机是否可飞越该栅格 |
| 搜索状态 | 浮点 [0,1] | 已搜索置信度（0=未搜索，1=已完全覆盖） |
| 搜索优先级 | 浮点 | 该栅格的搜索紧迫程度，重点区域优先级高 |
| 最后搜索时间 | 时间戳 | 上次被无人机传感器覆盖的时刻 |
| 目标信息 | 列表 | 该栅格内发现的目标记录 |

### 搜索衰减机制

- 已搜索栅格的置信度随时间衰减（适用于目标可能移动的场景）
- 衰减公式：`confidence(t) = confidence(t0) × exp(-λ × (t - t0))`
- λ 为衰减系数，可配置；静止目标场景可设为 0（不衰减）

### 动态更新接口

- 支持运行时更新单个或批量栅格的属性（如新增障碍物）
- 更新后自动通知路径规划模块检查受影响路径

## 六、无人机状态建模

### 无人机状态向量

```
UAV_State = {
    id:             无人机唯一标识
    position:       当前栅格坐标 (x, y)
    velocity:       当前速度 (m/s)
    heading:        航向角 (°)
    battery:        剩余电量百分比 (%)
    sensor_radius:  传感器覆盖半径 (栅格数)
    status:         当前状态枚举
    current_task:   当前执行的任务ID
    path:           当前路径点序列
}
```

### 状态枚举

- `IDLE`：待命
- `SEARCHING`：执行搜索任务
- `CONFIRMING`：抵近确认目标
- `RETURNING`：返航（低电量或任务完成）
- `AVOIDING`：临时避障中

### 电量模型（简化）

- 飞行耗电按匀速巡航建模：`consumption = distance / speed × power_rate`
- 当电量低于阈值（默认 20%）时强制返航
- 任务分配时需验证：完成任务后剩余电量足够返回起飞点

### 能力参数（同构默认值）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大速度 | 10 m/s | 巡航飞行速度 |
| 传感器覆盖半径 | 2 格（20m） | 飞越时可覆盖的周围栅格 |
| 续航时间 | 30 min | 满电飞行时长 |
| 返航电量阈值 | 20% | 触发强制返航 |

## 七、搜索任务建模

### 任务层次结构

```
搜索任务体系
├── 全局搜索任务（整个区域的覆盖搜索）
│   ├── 子区域任务（将区域划分为若干子区域）
│   │   └── 栅格搜索点（子区域内的具体航路点）
│   └── ...
└── 目标确认任务（发现目标后触发）
```

### 子区域划分策略

- 将全局可搜索区域按连通性和面积均衡原则划分为 N 个子区域（N 与无人机数量相关）
- 划分算法：基于 K-Means 或 Voronoi 图，以各子区域面积和无人机到达代价为优化目标
- 禁飞区和障碍物自然形成子区域边界

### 任务数据结构

```
Task = {
    id:             任务唯一标识
    type:           SEARCH / CONFIRM
    priority:       优先级（重点区域任务 > 普通区域 > 已衰减区域）
    target_cells:   目标栅格集合
    entry_point:    建议进入点坐标
    estimated_cost: 预估完成代价（飞行距离或时间）
    status:         PENDING / ASSIGNED / IN_PROGRESS / COMPLETED
    assigned_uav:   分配的无人机ID（可为空）
}
```

### 任务生成规则

- 系统启动时，根据子区域划分生成初始搜索任务集
- 重点区域生成高优先级任务
- 目标发现事件触发生成 CONFIRM 类型任务，优先级最高
- 已完成区域若搜索置信度衰减到阈值以下，重新生成搜索任务

### 子区域内搜索路径模式

- 对每个子区域，采用往返覆盖（Boustrophedon）模式生成航路点序列
- 考虑传感器覆盖半径，相邻航线间距 = 2 × sensor_radius（确保无缝覆盖）

## 八、任务分配算法设计

### 算法选型

Sequential Single-Item Auction（顺序单物品拍卖）

### 拍卖流程

```
1. 中心节点从任务池中取出待分配任务，按优先级排序
2. 对每个待分配任务，广播拍卖请求
3. 每架空闲或可抢占的无人机计算竞标值（bid）
4. 中心节点选择最优竞标者，分配任务
5. 更新分配状态，进入下一个任务的拍卖
6. 重复直到所有任务分配完毕或无可用无人机
```

### 竞标值计算

```
bid(uav_i, task_j) = w1 × distance_cost
                   + w2 × battery_cost
                   + w3 × priority_bonus
                   + w4 × load_balance_penalty
```

| 分量 | 说明 |
|------|------|
| distance_cost | 无人机当前位置到任务进入点的路径距离 |
| battery_cost | 完成任务所需电量占剩余电量的比例 |
| priority_bonus | 高优先级任务给出负值加成（鼓励竞标） |
| load_balance_penalty | 已分配任务多的无人机增加惩罚项 |

权重 w1–w4 可配置，默认以距离代价为主导。竞标值越低越优（代价最小的无人机获胜）。

### 特殊规则

- CONFIRM 任务：直接分配给发现目标的无人机，不走拍卖流程
- 电量不足以完成任务的无人机不参与竞标
- 当无人机从 CONFIRMING 状态恢复时，重新参与下一轮拍卖

## 九、路径规划算法设计

### 算法选型

A* 搜索算法

### 基本配置

- 搜索空间：二维栅格地图
- 移动模型：8 邻域（上下左右 + 四个对角方向）
- 代价函数：直线移动代价 1.0，对角移动代价 1.414
- 启发函数：对角距离（Diagonal Distance），保证可采纳性

### 代价增强

在基础移动代价上叠加环境因素：

```
actual_cost(cell) = base_move_cost
                  + obstacle_proximity_penalty
                  + priority_area_bonus
```

| 增强项 | 说明 |
|--------|------|
| obstacle_proximity_penalty | 靠近障碍物的栅格增加代价，引导路径与障碍物保持安全距离 |
| priority_area_bonus | 经过未搜索重点区域时给予负代价（奖励），鼓励路径顺带覆盖 |

### 路径后处理

- 路径平滑：对 A* 输出的栅格路径进行拐角平滑，减少不必要的转向
- 路径简化：移除共线的中间点，降低路径点数量
- 安全校验：确保平滑后路径不穿越障碍物或禁飞区

### 性能保障

- 200×200 栅格上单次 A* 搜索典型耗时 < 50ms（Python 实现）
- 若出现性能瓶颈，可选优化：
  - 限制搜索范围（局部 A*，设定搜索半径）
  - 使用 Jump Point Search（JPS）加速
  - 预计算关键节点间路径缓存

## 十、动态重规划机制

### 触发条件

| 触发事件 | 响应动作 | 优先级 |
|----------|----------|--------|
| 发现目标 | 发现者切换为 CONFIRMING，暂停当前搜索任务 | 最高 |
| 无人机离线/故障 | 回收其任务，重新拍卖分配给其他无人机 | 高 |
| 地图动态更新（新障碍物） | 检查所有路径是否受影响，受影响路径重新规划 | 高 |
| 电量低于阈值 | 该无人机切换为 RETURNING，其未完成任务重新分配 | 高 |
| 固定周期到达 | 更新搜索进度，评估是否需要调整任务分配 | 常规 |
| 目标确认完成 | 发现者恢复 IDLE，重新参与任务拍卖 | 常规 |

### 重规划策略

```
事件到达
  → 判断影响范围
    → 仅影响单架无人机路径：局部重规划（仅该无人机重算路径）
    → 影响任务分配格局：全局重规划（重新执行拍卖分配）
    → 影响地图结构：路径有效性批量校验 + 受影响路径重算
```

### 防抖机制

- 短时间内多个事件触发时，合并处理（如 200ms 内的多个地图更新批量处理）
- 避免频繁全局重规划导致无人机"抖动"（不断改变方向）

### 任务连续性保障

- 重规划后，若无人机当前任务未被取消，尽量保持原任务不变，只更新路径
- 任务切换时记录进度断点，后续可从断点继续

## 十一、冲突检测机制

### 冲突类型

| 冲突类型 | 定义 | 危险等级 |
|----------|------|----------|
| 空间冲突 | 两架无人机在同一时刻占据同一栅格或相邻栅格 | 高 |
| 路径交叉 | 两架无人机路径在时间窗口内存在交叉点 | 中 |
| 任务重叠 | 两架无人机被分配了覆盖范围重叠的搜索区域 | 低 |

### 检测时机

- 路径规划完成后，提交前进行冲突校验
- 每个决策周期对所有无人机当前路径做全局冲突扫描

### 检测算法

```
对每对无人机 (i, j)：
    将路径按时间步展开为 (位置, 时刻) 序列
    对每个时刻 t：
        if distance(pos_i(t), pos_j(t)) < safety_distance:
            标记为冲突，记录冲突时刻和位置
```

- 安全距离默认为 2 个栅格（20m）
- 时间步按无人机速度和栅格大小计算

### 冲突消解策略

| 策略 | 适用场景 | 方法 |
|------|----------|------|
| 时间偏移 | 路径交叉但时间可错开 | 让优先级低的无人机在冲突点前悬停等待 |
| 路径偏移 | 空间冲突且时间无法错开 | 优先级低的无人机绕行（重算路径，将对方路径临时标记为障碍） |
| 高度分层 | 多架无人机需通过同一狭窄区域 | 预留设计接口，当前二维模型暂不实现 |

### 优先级规则

- CONFIRMING > SEARCHING > RETURNING > IDLE
- 同状态下，电量更低的优先（确保其能完成任务后安全返航）

### 模块间通信

- 模块间通过 Python 方法调用直接通信（单进程架构）
- 统一使用数据类（dataclass）传递结构化信息
- 事件管理器采用观察者模式，模块可注册事件监听

### 主循环伪代码

```python
while simulation_running:
    # 1. 收集状态
    uav_states = fleet.get_all_states()
    events = event_manager.poll_events()

    # 2. 处理事件（高优先级立即响应）
    for event in events:
        if event.priority == HIGH:
            handle_urgent_event(event)

    # 3. 固定周期决策
    if time_for_regular_cycle():
        task_manager.update_tasks(map, uav_states)
        allocator.allocate(task_manager.pending_tasks, fleet)
        planner.plan_paths(fleet, map)
        planner.resolve_conflicts(fleet)

    # 4. 执行与仿真推进
    simulator.step(fleet)
    visualizer.update(map, fleet)
```

## 十四、开发阶段规划

| 阶段 | 内容 | 预期产出 | 建议周期 |
|------|------|----------|----------|
| **一期：基础框架** | 栅格地图建模、无人机状态模型、基本仿真循环 | 地图加载可视化、无人机可在地图上移动 | 1 周 |
| **二期：搜索任务** | 区域划分、任务生成、覆盖路径模式 | 单架无人机可完成子区域覆盖搜索 | 1 周 |
| **三期：路径规划** | A* 实现、路径平滑、障碍物绕行 | 无人机可规划并执行无碰撞路径 | 1 周 |
| **四期：多机协同** | 拍卖任务分配、冲突检测与消解 | 多架无人机协同搜索不冲突 | 1.5 周 |
| **五期：动态响应** | 事件驱动重规划、目标发现响应、电量管理 | 系统可对突发事件动态调整 | 1 周 |
| **六期：评估优化** | 评价指标实现、参数调优、多场景验证 | 完整实验数据与分析报告 | 1 周 |

### 各阶段原则

- 每阶段结束时有可运行的仿真演示
- 后一阶段建立在前一阶段完整功能之上
- 每阶段附带对应单元测试

## 十五、评价指标设计

### 覆盖率指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 全局覆盖率 | 已搜索栅格数 / 可搜索栅格总数 × 100% | 核心指标，反映搜索完成度 |
| 重点区域覆盖率 | 重点区域已搜索栅格数 / 重点区域栅格总数 × 100% | 反映任务优先级执行效果 |
| 覆盖均匀度 | 各子区域覆盖率的标准差 | 越小说明分配越均衡 |

### 时间效率指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 全覆盖耗时 | 达到目标覆盖率（如 95%）所需时间步数 | 越少越好 |
| 重点区域首覆盖时间 | 重点区域达到 100% 覆盖的时刻 | 反映优先级响应速度 |
| 目标发现平均延迟 | 目标出现到被发现的平均时间间隔 | 反映搜索密度与效率 |

### 协同效率指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 重复覆盖率 | 被多次搜索的栅格数 / 已搜索栅格数 | 越低说明协同越好，减少冗余 |
| 任务分配均衡度 | 各无人机执行任务数量的标准差 | 越小越均衡 |
| 飞行总路程利用率 | 有效搜索路程 / 总飞行路程 | 越高说明路径越高效 |

### 安全性指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 冲突发生次数 | 仿真中检测到的空间冲突总次数 | 应为 0 |
| 禁飞区侵入次数 | 路径穿越禁飞区的事件数 | 应为 0 |
| 最低电量裕度 | 所有无人机返航时的最低剩余电量 | 应高于安全阈值 |

### 动态响应指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 重规划平均耗时 | 触发重规划到新路径生成的平均延迟 | 应 < 1秒 |
| 目标确认响应时间 | 从发现目标到发现者开始抵近的时间 | 越短越好 |
| 任务恢复时间 | 无人机从异常状态恢复后重新获得任务的平均时间 | 反映系统恢复能力 |
